# AI Research Brief Workflow

Write an AI Intelligence Brief about the runtime topic.

The runtime topic must dominate the brief. Use Hermes soul only as behavior context, not as the research subject unless the topic explicitly asks about Hermes.

If structured source items are provided, ground the brief in those items and do not invent additional sources.

Required disclaimer when no structured source items are provided:
"This is model-generated analysis based on the provided topic and local context, not live web research."

Required disclaimer when structured source items are provided:
"This is source-grounded analysis based on provided source items and local context, not live web research."

Required format:

## Title

Use a concise title.

## Disclaimer

Include the correct required disclaimer exactly.

## Executive Summary

Give a 3 bullet summary. Each bullet should explain significance, not just restate facts.

## Top Signals

List up to 5 signals.

For each signal include:

- Signal
- Why it matters
- Technology map
- Ecosystem impact
- Recommended action
- Confidence

Allowed recommended actions:

- Ignore
- Watch
- Read
- Prototype
- Implement
- Review later

## Intelligence Score

Provide a 0 to 100 score for the topic's current importance.

Explain the score using these dimensions:

- Importance
- Impact
- Momentum
- Engineering Value
- Adoption
- Longevity
- Novelty

Do not treat the score as a simple average. Explain the main weighting reason.

## Recommended Next Step

End with one practical next step for the user.

If evidence is insufficient, say what would need to be verified before taking stronger action.
